# MCヤスくん セットアップガイド

> CS_HUBくんをベースにした、あなた専用のタスク自動管理ボットです。
> Chatworkに届くメッセージを自動でタスク化し、ダッシュボードで一元管理できます。

---

## どんなシステム？

```
あなたのChatwork（メンション付きメッセージ）
         ↓
   MCヤスくん（Chatworkボット）
         ↓ 10分ごとに自動チェック
   管理ダッシュボード（Web）
         ├── タスク自動生成
         ├── 担当者アサイン
         ├── 期限リマインド送信
         └── 進捗管理
```

---

## 準備するもの（5つ）

### 1. Chatwork専用アカウント（必須）
「MCヤスくん」専用のChatworkアカウントを1つ新規作成してください。
このアカウントがボットとして動作します。

### 2. Chatwork APIトークン（必須）
MCヤスくんアカウントでログインして取得します。

1. Chatwork右上プロフィール → 「サービス連携」
2. 「APIトークン発行」→ 表示された文字列をメモ

### 3. MCヤスくんのアカウントID（必須）
1. MCヤスくんアカウントでChatworkにログイン
2. 右上プロフィール → プロフィール画面のURLを確認
3. URL内の数字（例: `12345678`）がアカウントID

### 4. 監視したいChatworkルームのID（必須）
通知を受け取りたいグループルームのIDです。

1. 監視したいルームを開く
2. URLの `rid=` 以降の数字（例: `987654`）がルームID
3. 複数ある場合はカンマ区切りでまとめる（例: `987654,111222`）

### 5. スタッフ名リスト（必須）
担当者として登録するスタッフの名前リスト（カンマ区切り）
例: `ヤス,田中,鈴木`

## セットアップ手順

### STEP 1: GitHubにコードを複製する

1. [GitHub](https://github.com) にログイン（アカウントがない場合は無料で作成）
2. このリポジトリ `https://github.com/tfunakita/cs_hub` を開く
3. 右上の「Fork」ボタンを押す → 自分のアカウントにコピーされる

### STEP 2: Railwayでサーバーを立てる

1. [Railway](https://railway.app) にGitHubアカウントでログイン（無料）
2. 「New Project」→「Deploy from GitHub repo」
3. STEP 1 でForkしたリポジトリを選択
4. デプロイが始まる（最初はエラーでもOK、次のステップで解消）

### STEP 3: データ保存領域を追加する

1. Railway → 作ったプロジェクト → 「Add Volume」
2. Mount Path に `/data` と入力 → 追加

### STEP 4: 環境変数を設定する

Railwayのプロジェクト → 「Variables」タブで以下を1つずつ追加：

| 変数名 | 値 | 説明 |
|--------|----|----|
| `CHATWORK_API_TOKEN` | 手順2で取得したトークン | |
| `CHATWORK_HUB_ACCOUNT_ID` | 手順3で確認したID | |
| `CHATWORK_ROOM_IDS` | 手順4で確認したルームID | 複数の場合はカンマ区切り |
| `STAFF_NAMES` | スタッフ名 | 例: `ヤス,田中,鈴木` |
| `DB_PATH` | `/data/cs_hub.db` | そのままコピー |
| `DEFAULT_ASSIGNEE` | 自動アサインしたい担当者名（任意） | |
| `AI_SUMMARY_ENABLED` | `false` | AI要約不要なら false |

全部入力したら自動でサーバーが再起動します。

### STEP 5: MCヤスくんをルームに招待する

1. 監視したいChatworkグループルームを開く
2. MCヤスくんアカウントをメンバーとして招待

### STEP 6: 動作確認

1. Railwayのダッシュボードに表示されているURLを開く（例: `https://xxxxx.up.railway.app`）
2. ダッシュボードが表示されればOK
3. 監視ルームでMCヤスくんにメンションを送る：
   ```
   [To:MCヤスくんのアカウントID] MCヤスくん
   テスト送信です。
   ```
4. 10分以内にダッシュボードにタスクが現れれば完了

## 使い方

### タスクを作る（Chatworkから自動）

MCヤスくんにメンションするだけ。
```
[To:アカウントID] MCヤスくん
〇〇の件、対応お願いします。
```
→ 自動でタスクになります。

### ダッシュボードの見方

| 画面 | 内容 |
|------|------|
| ダッシュボード | KPI・未対応アラート・スタッフ別状況 |
| 振り分け | 担当者が未定のタスク一覧 |
| タスク一覧 | 全タスクの一覧・検索・絞り込み |
| カンバン | ステータス別のカード表示 |
| 設定 | スタッフ追加・定期タスク設定 |

### ステータスの意味

| ステータス | 意味 |
|-----------|------|
| open | 未着手 |
| in_progress | 対応中 |
| done | 完了 |
| closed | クローズ（完了連絡済み） |

## よくある質問

**Q. タスクが自動生成されない**
- MCヤスくんを監視ルームに招待しているか確認
- `CHATWORK_ROOM_IDS` のルームIDが正しいか確認
- 10分待ってから再確認（ポーリング間隔が10分）

**Q. ダッシュボードにアクセスできない**
- Railwayのデプロイが完了しているか確認（Variables設定後に自動再起動）
- `DB_PATH=/data/cs_hub.db` が設定されているか確認

**Q. 通知が届かない**
- スタッフへのDM通知はオプション機能（`STAFF_ROOM_名前=ルームID` を設定すると有効）
